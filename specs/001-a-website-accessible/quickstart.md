# Quickstart: Personalized Content Dashboard

Path: /home/josh/projects/joshNews/specs/001-a-website-accessible/quickstart.md

This quickstart walks through running the service locally for development and testing.

Prerequisites

- Bun (recommended): https://bun.sh
- Node-compatible shell (Linux/Mac)

Steps

1. Install dependencies
   - `bun install`
2. Run tests (unit + contract)
   - `bun test` # contract tests expect the server to be started by the test harness
3. Start the server
   - `bun src/main.ts` # defaults to PORT=3000
4. Basic API endpoints (examples)
   - `POST /api/v1/auth/magic-link` -> 202 Accepted
   - `GET /api/v1/auth/verify?token=...` -> 200 OK
   - `POST /api/v1/sources` -> 201 Created
   - `GET /api/v1/sources` -> 200 OK
   - `GET /api/v1/dashboards/daily` -> 200 OK (returns items ordered by score desc; capped at 50)

Local test helper

- The repository includes `tests/setup/server.ts` which exposes `startTestServer(port?)` and `stopTestServer()` to make contract tests self-contained.

Notes

- This quickstart assumes Postgres for production, but Phase 1 uses file/in-memory adapters for local dev and deterministic tests.
